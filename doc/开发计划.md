- 后端自动发现 USB 连接的 Android 设备，前端实时展示。

- 设备断开后 UI 正确更新，资源正确释放。

## 风险与应对

| 风险 | 影响 | 应对策略 |
| :--- | :---: | :--- |
| scrcpy 协议变更 | 高 | 锁定 scrcpy-server v3.3.4，关注上游更新 |
| WebCodecs 浏览器兼容性 | 中 | 保留 MJPEG/iOS Canvas 渲染路径；必要时实现 MSE 降级 |
| WDA 签名过期（7 天） | 高 | 文档要求有效开发者证书，失败时前端提示联系管理员 |
| iOS 低版本 WDA/tidevice 链路不稳定 | 高 | 进程树清理、自动重试、连续失败标记 `unavailable` |
| go-ios 对新 iOS 版本支持滞后 | 中 | 建立回退机制，后续评估 pymobiledevice3 tunnel |
| 多设备并发性能瓶颈 | 中 | 背压控制 + 最大并发数限制 + 独立会话资源清理 |
| Electron 打包资源路径错误 | 中 | 通过 `MSCA_RESOURCES_PATH` 与 `process.resourcesPath` 做端到端验证 |
