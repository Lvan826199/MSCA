## 架构调整说明

废弃 Electron 直接调用 scrcpy GUI 的方案，所有 scrcpy-server（v3.3.4）管理逻辑统一下沉到 Python 后端：

```
桌面端：Electron → BackendManager 启动本地后端 → 前端 WS 连接 127.0.0.1:18000
Web 端：浏览器 → 前端 WS 连接远程后端

共用：Python 后端（ScrcpyServerManager → adbutils → 设备）+ Vue 前端（WebCodecs → Canvas）
```

- M3+M5 合并为"Android 统一投屏"，M4+M6 合并为"Android 统一控制"
- `electron/scrcpy-manager.js` 废弃
- 前端代码零分支，桌面端/Web 端完全统一

### UI 方案决策：单页面多设备网格布局

> **决策日期**：2026-04-24

采用**单页面多设备网格布局**，而非弹出独立窗口，原因：

- **多设备同屏**：网格布局（2×2、3×3 等）可在一个页面同时展示所有设备画面
- **同步操作**：所有设备共享同一 Vue 实例状态，"同步模式"开关可将操作广播至所有设备
- **状态管理简单**：无需跨窗口通信，控制逻辑集中在单一上下文中

**页面路由结构**：

```
/              → HomeView（设备列表）
/mirror        → MirrorView（投屏视图，支持多设备网格）
```

**MirrorView 布局设计**：

```
MirrorView
├── 顶部工具栏：添加设备、同步模式开关、全部停止
├── 设备网格区（自适应列数）
│   ├── DeviceMirrorPanel（设备 1）：Canvas + 设备名 + 停止按钮
│   ├── DeviceMirrorPanel（设备 2）：Canvas + 设备名 + 停止按钮
│   └── ...
└── 底部控制栏：Home / Back / 电源键、文本输入框
```

**交互流程**：

1. 设备列表页点击"投屏"→ 路由跳转至 `/mirror?device=<id>`
2. MirrorView 中可继续点击"添加设备"追加更多设备到网格
3. 开启"同步模式"后，所有触控/按键操作广播至当前所有投屏设备
4. 单击某设备画面可放大至全屏，再次点击退出全屏

## 第三步：M4/M6 Android 统一控制 ✅ 已完成

**目标**：前端捕获触控/按键事件，通过 WS 发送至后端，后端编码为 scrcpy 协议写入 control_socket。

**前置条件**：M3/M5 完成。

> **ws-scrcpy 参考要点**：
> - 控制消息二进制编码格式直接参考 ws-scrcpy 的 `ControlMessage` 系列类（详见技术栈概览 §8.2）
> - 键盘映射参考 ws-scrcpy 的 `KeyToCodeMap`（浏览器 `KeyboardEvent.code` → Android keycode）
> - 触控坐标转换参考 `InteractionHandler.buildTouchOnClient` 的比例计算
> - 滚动事件 30ms 节流策略参考 ws-scrcpy 的 `SCROLL_EVENT_THROTTLING_TIME`
> - 剪贴板同步参考 ws-scrcpy 的 TYPE=8/9 + 设备消息回传机制

### 当前已完成

全部控制功能已实现（S3.1~S3.9）：
- `backend/app/websocket/control.py` — 控制 WS 端点，支持 touch/key/text/scroll/back/home/power/通知栏/设置面板/剪贴板/旋转
- `frontend/src/composables/useDeviceControl.js` — Canvas 鼠标/触摸/键盘事件捕获、坐标转换、键盘映射（80+ 键位）、metaState 修饰键
- `frontend/src/components/DeviceControlBar.vue` — 全功能控制栏（导航键/音量/电源/通知栏/设置/旋转/文本输入/剪贴板）
- `backend/app/scrcpy/protocol.py` — scrcpy 二进制协议编码（全部 12 种控制类型）

**目标**：支持多台设备同时投屏控制，实现同步操作模式，优化多设备场景下的资源管理。

**前置条件**：M4/M6 完成。

| S4.2 | 设备选择器弹窗 | `frontend/src/components/DeviceSelector.vue`（新建） | 从已连接设备列表中选择要添加到投屏网格的设备 |
| S4.3 | 网格布局优化 | `frontend/src/views/MirrorView.vue` | 自适应列数（1/2/3/4列），支持拖拽调整设备顺序 |
| S4.4 | 后端并发资源管理 | `backend/app/core/device_manager.py` | 多设备同时投屏时的资源隔离、异常不互相影响 |
| S4.5 | 全部停止优化 | `frontend/src/views/MirrorView.vue` + 后端 | 批量停止所有投屏，确保资源完全释放不卡住 |

### 验收标准

- 可同时投屏 4+ 台设备，各设备独立控制互不干扰

- 全部停止按钮可在 3 秒内释放所有资源
- 设备断开后自动从网格移除，不影响其他设备
