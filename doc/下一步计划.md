```
共用：Python 后端（ScrcpyServerManager → adbutils → 设备）+ Vue 前端（WebCodecs → Canvas）
```

- M3+M5 合并为"Android 统一投屏"，M4+M6 合并为"Android 统一控制"
- `electron/scrcpy-manager.js` 废弃
- 前端代码零分支，桌面端/Web 端完全统一

### UI 方案决策：单页面多设备网格布局

> **决策日期**：2026-04-24

采用**单页面多设备网格布局**，而非弹出独立窗口，原因：

- **多设备同屏**：网格布局（2×2、3×3 等）可在一个页面同时展示所有设备画面
- **同步操作**：所有设备共享同一 Vue 实例状态，"同步模式"开关可将操作广播至所有设备
- **状态管理简单**：无需跨窗口通信，控制逻辑集中在单一上下文中

**页面路由结构**：

```
/              → HomeView（设备列表）
/mirror        → MirrorView（投屏视图，支持多设备网格）
```

**MirrorView 布局设计**：

```
MirrorView
├── 顶部工具栏：添加设备、同步模式开关、全部停止
├── 设备网格区（自适应列数）
│   ├── DeviceMirrorPanel（设备 1）：Canvas + 设备名 + 停止按钮
│   ├── DeviceMirrorPanel（设备 2）：Canvas + 设备名 + 停止按钮
│   └── ...
└── 底部控制栏：Home / Back / 电源键、文本输入框
```

**交互流程**：

1. 设备列表页点击"投屏"→ 路由跳转至 `/mirror?device=<id>`
2. MirrorView 中可继续点击"添加设备"追加更多设备到网格
3. 开启"同步模式"后，所有触控/按键操作广播至当前所有投屏设备
4. 单击某设备画面可放大至全屏，再次点击退出全屏

## 第三步：M4/M6 Android 统一控制

**目标**：前端捕获触控/按键事件，通过 WS 发送至后端，后端编码为 scrcpy 协议写入 control_socket。

**前置条件**：M3/M5 完成。

### 任务清单

| 序号 | 任务 | 涉及文件 | 说明 |
| :---: | :--- | :--- | :--- |
| S3.1 | 控制指令 WS 端点 | `backend/app/websocket/control.py`（新建） | `/ws/control/{device_id}` 接收 JSON 指令 |
| S3.2 | 触控事件捕获与坐标转换 | `frontend/src/composables/useControlSender.js`（新建） | Canvas 鼠标事件 → 设备坐标 → WS 发送 |
| S3.3 | 控制栏组件 | `frontend/src/components/ControlBar.vue`（新建） | Home/Back/电源键、文本输入框 |
| S3.4 | 文本输入与屏幕旋转 | 后端 scrcpy 协议 + 前端 UI | 文本注入、横竖屏自适应 |

### 验收标准

- Canvas 上点击/滑动操作正确映射到设备
- 虚拟按键（Home、Back、电源）响应正确
- 可输入中英文文本
- 设备旋转后画面比例自适应
