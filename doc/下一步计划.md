## 架构调整说明

废弃 Electron 直接调用 scrcpy GUI 的方案，所有 scrcpy-server（v3.3.4）管理逻辑统一下沉到 Python 后端：

```
桌面端：Electron → BackendManager 启动本地后端 → 前端连接本地动态端口（从 18000 起探测，写入 .backend-port）
Web 端：浏览器 → 前端 WS 连接远程后端

共用：Python 后端（ScrcpyServerManager → adbutils → 设备）+ Vue 前端（WebCodecs → Canvas）
```

- M3+M5 合并为"Android 统一投屏"，M4+M6 合并为"Android 统一控制"
- `electron/scrcpy-manager.js` 废弃
- 前端代码零分支，桌面端/Web 端完全统一

### M13.5 Windows Electron 打包

**任务**：配置 electron-builder，确保前端、后端、二进制工具全部正确打包。

**关键配置**：
```json
{
  "build": {
    "extraResources": [
      { "from": "resources/", "to": "resources/" },
      { "from": "bin/", "to": "bin/" }
    ]
  }
}
```

**验收标准**：
- `npm run electron:build` 生成 `dist/electron/MSCA Setup 0.1.0.exe`
- 安装后应用可正常启动，自动启动内嵌后端
- 设备列表正常显示，投屏功能可用

### M13.6 + M13.8 二进制工具内嵌

**任务**：将 scrcpy-server、go-ios 等二进制工具打包进 Electron 应用。

**需要内嵌的文件**：
- `bin/android/scrcpy-server` — scrcpy-server v3.3.4
- `bin/android/adb.exe` — ADB 工具（如需）
- `bin/ios/ios.exe` — go-ios 可执行文件
- `resources/msca-backend/` — Nuitka standalone 后端运行时目录

**路径解析**：Electron 打包后 `extraResources` 位于 `process.resourcesPath`，需确保 BackendManager 和 DeviceManager 能正确解析打包后的路径。
