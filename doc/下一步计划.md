## 第一步：M2 Android 设备发现与管理

**目标**：通过 adbutils 实现 Android 设备的自动发现、状态监控与前端展示。

### 任务清单

| 序号 | 任务 | 涉及文件 | 说明 |
| :---: | :--- | :--- | :--- |
| S1.1 | ADB 设备列表获取 | `backend/app/api/devices.py`、`backend/app/core/device_manager.py` | 使用 adbutils 获取已连接设备列表，返回设备 ID、型号、Android 版本、分辨率、连接状态 |
| S1.2 | DeviceInfo 统一数据模型 | `backend/app/models/device.py`（新建） | 定义 Pydantic 模型，Android/iOS 共用，包含 id、platform、model、version、resolution、status 等字段 |
| S1.3 | 设备插拔监听 | `backend/app/core/device_manager.py` | 后台轮询（间隔 2~3 秒）检测设备变化，维护内存设备列表 |
| S1.4 | 设备变化 WebSocket 推送 | `backend/app/websocket/handler.py` | 新增 `/ws/devices` 端点，设备列表变化时主动推送至所有已连接客户端 |
| S1.5 | 前端设备列表 UI | `frontend/src/views/HomeView.vue`、`frontend/src/components/DeviceCard.vue`（新建） | 设备卡片组件：展示型号、状态指示灯、分辨率、操作按钮（投屏/断开） |
| S1.6 | 前端设备列表 WebSocket 集成 | `frontend/src/composables/useDevices.js`（新建） | 订阅 `/ws/devices`，响应式维护设备列表状态 |

### 验收标准

- `GET /api/devices` 返回当前所有已连接 Android 设备信息
- USB 插拔设备后，前端设备列表在 3 秒内自动更新
- 设备卡片正确展示型号、版本、状态

---

## 第二步：M3 Android 桌面端投屏（Electron + Scrcpy）

**目标**：通过 Electron 子进程管理 scrcpy，实现桌面端 Android 设备投屏。

**前置条件**：M2 完成，设备列表可用。

### 任务清单

| 序号 | 任务 | 涉及文件 | 说明 |
| :---: | :--- | :--- | :--- |
| S2.1 | ScrcpyManager 完整实现 | `electron/scrcpy-manager.js` | 实现 scrcpy 子进程的启动、停止、参数配置（分辨率/码率/帧率） |
| S2.2 | scrcpy 路径检测 | `electron/scrcpy-manager.js` | 自动检测系统 PATH 中的 scrcpy，或使用内嵌路径 |
| S2.3 | 投屏 IPC 通道 | `electron/main.js`、`electron/preload.js` | 新增 `start-mirroring`、`stop-mirroring` IPC 接口 |
| S2.4 | 前端投屏触发 | `frontend/src/views/HomeView.vue`、`frontend/src/components/DeviceCard.vue` | 设备卡片点击"投屏"按钮，通过 electronAPI 调用 scrcpy |
| S2.5 | 进程异常恢复 | `electron/scrcpy-manager.js` | scrcpy 崩溃检测与用户提示 |
| S2.6 | 设备断开处理 | `electron/scrcpy-manager.js` | USB 断开时自动终止对应 scrcpy 进程，更新 UI 状态 |

### 验收标准

- 点击设备卡片"投屏"按钮，scrcpy 窗口弹出并显示设备画面
- 投屏参数（分辨率、码率）可配置
- 设备断开后 scrcpy 进程正确释放，UI 状态更新
- scrcpy 异常退出后给出用户提示

---

## 第三步：M4 Android 桌面端控制

**目标**：通过 scrcpy 原生控制能力实现设备操控（触控、按键、文本输入）。

**前置条件**：M3 完成，投屏窗口可用。

### 任务清单

| 序号 | 任务 | 涉及文件 | 说明 |
| :---: | :--- | :--- | :--- |
| S3.1 | 触控事件透传 | scrcpy 原生支持 | scrcpy 窗口内鼠标事件自动映射为设备触控，验证点击/滑动 |
| S3.2 | 按键映射配置 | `electron/scrcpy-manager.js` | 配置 scrcpy 快捷键：Home、Back、电源键、音量键 |
| S3.3 | 文本输入 | `electron/scrcpy-manager.js` | 通过 scrcpy 剪贴板注入实现文本输入 |
| S3.4 | 屏幕旋转处理 | scrcpy 原生支持 | 验证设备横竖屏切换时 scrcpy 窗口自适应 |

### 验收标准

- scrcpy 窗口内可进行点击、滑动操作
- 快捷键可触发 Home、Back、电源键
- 可向设备输入中英文文本
- 设备旋转后画面比例正确

---

## 执行顺序与依赖关系

```
M2（设备发现） ──→ M3（桌面端投屏） ──→ M4（桌面端控制）
  S1.1~S1.6         S2.1~S2.6            S3.1~S3.4
```

完成 M2~M4 后，第一阶段（Android 桌面端 MVP）即告完成，可进入第二阶段（Android Web 端 + 多设备）。
