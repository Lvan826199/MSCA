## 1. 开发环境搭建

### 1.1 前置依赖

| 工具 | 版本要求 | 用途 |
| :--- | :--- | :--- |
| Node.js | ≥18.x | 前端构建与 Electron 运行 |
| Python | ≥3.13 | 后端服务 |
| uv | 最新版 | Python 包管理与虚拟环境 |
| ADB (Platform Tools) | ≥33.0 | Android 设备通信 |
| Git | 最新版 | 版本控制 |

### 1.2 安装 Node.js

1. 访问 https://nodejs.org/ 下载 LTS 版本
2. 安装时勾选"Add to PATH"
3. 验证：`node --version` 和 `npm --version`

### 1.3 安装 Python 与 uv

1. 访问 https://www.python.org/downloads/ 下载 Python 3.13+
2. 安装时勾选"Add Python to PATH"
3. 安装 uv：

```bash
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

4. 验证：`python --version` 和 `uv --version`

### 1.4 安装 ADB (Android Platform Tools)

ADB 是与 Android 设备通信的核心工具，需要单独安装。

#### Windows

1. 访问 https://developer.android.com/tools/releases/platform-tools 下载 Windows 版本
2. 解压到固定目录，例如 `D:\android_sdk\platform-tools`
3. 将该目录添加到系统环境变量 PATH：
   - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   - 在"系统变量"中找到 `Path`，点击编辑
   - 新建一行，填入 `D:\android_sdk\platform-tools`
   - 确定保存
4. 重新打开终端，验证：`adb version`

#### macOS

```bash
# 方式一：Homebrew
brew install android-platform-tools

# 方式二：手动下载
# 访问上述链接下载 macOS 版本，解压后将路径加入 ~/.zshrc：
# export PATH="$PATH:/path/to/platform-tools"
```

验证：`adb version`，应输出类似 `Android Debug Bridge version 1.0.41`。

#### USB 驱动（Windows）

部分 Android 设备在 Windows 上需要安装 USB 驱动：

- 小米/Redmi：安装小米助手或从设备管理器更新驱动
- 三星：安装 Samsung USB Driver
- 通用方案：安装 Google USB Driver（通过 Android SDK Manager）

如果 `adb devices` 显示设备但状态为 `unauthorized` 或 `offline`，通常是驱动问题。

## 2. Android 设备连接

### 2.1 开启 USB 调试

1. 进入手机"设置" → "关于手机"
2. 连续点击"版本号"7 次，开启开发者选项
3. 返回"设置" → "开发者选项"（部分手机在"更多设置"中）
4. 开启以下选项：
   - **USB 调试**：必须开启
   - **USB 安装**：建议开启（部分操作需要）
   - **USB 调试（安全设置）**：小米设备建议开启

### 2.2 连接设备

1. 使用 USB 数据线连接手机与电脑
2. 手机弹出"允许 USB 调试"对话框，勾选"始终允许"并确认
3. 验证连接：

```bash
adb devices
```

正常输出示例：

```
List of devices attached
e63590450406    device
```

状态说明：

| 状态 | 含义 | 处理方式 |
| :--- | :--- | :--- |
| `device` | 正常连接 | 无需处理 |
| `unauthorized` | 未授权 | 在手机上确认 USB 调试授权弹窗 |
| `offline` | 设备离线 | 重新插拔 USB 或执行 `adb kill-server && adb devices` |
| 无设备显示 | 未识别 | 检查 USB 线、驱动、USB 调试开关 |

### 2.3 获取设备信息

```bash
# 设备型号
adb shell getprop ro.product.model

# Android 版本
adb shell getprop ro.build.version.release

# 屏幕分辨率
adb shell wm size
```

## 4. 启动开发环境

### 4.1 一键启动（推荐）

```bash
npm run electron:dev
```

该命令会自动完成以下操作，无需手动干预：
1. 启动 Vite 前端开发服务器（localhost:5173）
2. 启动 Python 后端服务（localhost:18000）
3. 等待前端就绪后打开 Electron 桌面窗口

> 这是最简单的启动方式，适合日常开发和不熟悉命令行的用户。

### 4.2 仅启动 Web 前端

```bash
npm run dev
```

浏览器访问 http://localhost:5173 查看界面。需要单独启动后端服务才能使用完整功能。

### 4.3 单独启动后端（调试用）

```bash
cd backend
uv run uvicorn app.main:app --reload --host 127.0.0.1 --port 18000
```

验证：访问 http://127.0.0.1:18000/health ，应返回 `{"status":"ok"}`。

### 4.4 开发模式选择

| 场景 | 启动方式 | 说明 |
| :--- | :--- | :--- |
| 日常开发（推荐） | `npm run electron:dev` | 一键启动，自动管理前端和后端 |
| 纯前端开发 | `npm run dev` | 浏览器访问，热更新快 |
| 后端调试 | `uv run uvicorn ...` | 单独调试 API 和 WebSocket |

## 5. 构建与打包

### 5.1 前端构建

```bash
npm run build
```

预期结果：生成 `frontend/dist/` 目录，无报错。

### 5.2 后端编译（Nuitka）

将 Python 后端编译为独立可执行文件，供 Electron 桌面端内嵌使用：

```bash
npm run backend:build
```

预期结果：生成 `resources/msca-backend.exe`。

### 5.3 验证后端编译产物

编译完成后，验证 exe 是否能正常启动和响应请求：

```bash
npm run backend:verify
```

该脚本会自动：
1. 启动 `resources/msca-backend.exe`
2. 检查 `/health` 健康端点
3. 检查 `/api/devices` 设备列表接口
4. 输出验证结果并自动清理进程

### 5.4 完整打包（Electron 安装包）

```bash
npm run electron:build
```

该命令会依次执行：前端构建 → 后端 Nuitka 编译 → Electron 打包。

预期结果：生成 `dist-electron/` 目录，包含 `MSCA-Setup.exe` 安装包。

### 5.5 命令速查

| 命令 | 说明 |
| :--- | :--- |
| `npm run electron:dev` | 一键启动桌面端开发模式 |
| `npm run dev` | 仅启动前端开发服务器 |
| `npm run build` | 构建 Vue 前端 |
| `npm run backend:build` | Nuitka 编译后端为 exe |
| `npm run backend:verify` | 验证后端 exe 是否正常 |
| `npm run electron:build` | 完整打包 |

## 6. 常见问题

### Q: `adb devices` 无输出

- 检查 USB 线是否支持数据传输（部分线只能充电）
- 检查手机 USB 调试是否开启
- Windows：检查设备管理器中是否有未识别设备，安装对应驱动
- 执行 `adb kill-server && adb start-server` 重启 ADB 服务

### Q: 后端启动报错 `ModuleNotFoundError`

```bash
cd backend
uv sync  # 重新安装依赖
```

### Q: Electron 启动后白屏

- 使用 `npm run electron:dev` 一键启动即可，它会自动启动前端和后端
- 如果仍然白屏，检查终端输出是否有报错信息

### Q: 端口 18000 被占用

后端会自动尝试 18001、18002。如需手动指定：

```bash
uv run uvicorn app.main:app --reload --host 127.0.0.1 --port 18001
```
