# MSCA 设备部署指南

> 指导用户完成 iOS WDA 的打包签名安装与 Android 设备的连接准备。
> 参考资料:[Sonic iOS 部署文档](https://soniccloudorg.github.io/deploy/ios-deploy.html)、[Sonic Android 部署文档](https://soniccloudorg.github.io/deploy/android-deploy.html)

---

## 一、iOS 设备部署(WebDriverAgent)

MSCA 的 iOS 投屏与控制依赖设备上运行的 WebDriverAgent(WDA)。WDA 必须用 Apple 开发者证书签名后安装到设备,这一步需要一台 **macOS + Xcode** 环境(仅打包时需要,日常使用 Windows 即可)。

### 1.1 获取 WDA 源码

```bash
git clone https://github.com/appium/WebDriverAgent.git
cd WebDriverAgent
```

### 1.2 Xcode 签名配置

1. 用 Xcode 打开 `WebDriverAgent.xcodeproj`
2. 分别选中 **WebDriverAgentLib** 和 **WebDriverAgentRunner** 两个 Target:
   - `Signing & Capabilities` → 勾选 `Automatically manage signing`
   - `Team` 选择你的开发者账号
3. 修改 **WebDriverAgentRunner** 的 `Bundle Identifier` 为自有标识(避免与他人冲突),**建议保留 `.xctrunner` 后缀**,例如 `com.yourcompany.wda.xctrunner`
   - MSCA 后端按 `backend/config/wda_config.json` 中的 `wda_bundle_id_pattern`(默认 `com.*.xctrunner`)自动检测 WDA;若你的 Bundle ID 不符合该模式,需在该文件中显式填写 `wda_bundle_id`

> **免费 vs 付费开发者账号**:
> - 免费 Apple ID 也可签名,但证书 **7 天过期**,到期后需重新签名安装;且同一免费账号最多同时签 3 个 App
> - 付费开发者账号($99/年)证书有效期 1 年,适合长期部署

### 1.3 直接构建到设备(开发机模式)

设备 USB 连接 Mac 后:

1. Xcode 顶部 Scheme 选择 **WebDriverAgentRunner**,目标设备选择你的 iPhone
2. 菜单 `Product → Test`(或 `Build` 后手动安装)
3. 首次运行会在设备上弹出"未受信任的开发者":到 iPhone `设置 → 通用 → VPN与设备管理` 中信任对应证书

### 1.4 导出 WDA.ipa(批量分发模式)

如需在多台设备/无 Mac 环境安装,可先导出 ipa:

```bash
# 在 WebDriverAgent 源码目录执行 Release 构建
xcodebuild -project WebDriverAgent.xcodeproj \
  -scheme WebDriverAgentRunner \
  -configuration Release \
  -destination "generic/platform=iOS" \
  CODE_SIGNING_ALLOWED=YES \
  build

# 找到产物 WebDriverAgentRunner-Runner.app(通常在 DerivedData 的 Build/Products/Release-iphoneos 下)
mkdir Payload
cp -r WebDriverAgentRunner-Runner.app Payload/
zip -r WDA.ipa Payload
```

得到的 `WDA.ipa` 可放入 MSCA 的 `resources/` 目录,通过以下任一方式安装:

```bash
# 方式一:MSCA 内置 go-ios(项目 bin/ios/ 下)
bin\ios\ios.exe install --path=WDA.ipa --udid=<设备UDID>

# 方式二:tidevice(iOS ≤15.x)
uv run tidevice -u <设备UDID> install WDA.ipa

# 方式三:MSCA 桌面端的应用安装功能(设备卡片 → 安装应用 → 选择 WDA.ipa)
```

> 注意:ipa 内的签名绑定了打包时指定的设备列表(开发证书)或企业分发范围。免费账号导出的 ipa 只能装到打包时注册过的设备上。

### 1.5 iOS 环境部署要求:WebDriverAgentRunner 打包策略

不同 iOS 大版本对 WebDriverAgentRunner 的运行时兼容性不同。WDA 能安装、能短暂启动,不代表能在设备上长期保持 `/status`、`/screenshot`、MJPEG 流服务稳定。部署时不要只维护一份 WDA.ipa,应按 iOS 系统版本分两类打包。

| 设备系统 | 推荐 WDA 包 | 推荐用途 | 说明 |
| :--- | :--- | :--- | :--- |
| iOS 15/16/17/18 | Appium WebDriverAgentRunner `9.0.6` 或当前已验证版本 | 现代设备主包 | 当前高版本设备已验证正常,继续使用现有 Xcode 16 / WDA 9.0.6 包即可 |
| iOS 13/14 | Appium WebDriverAgentRunner `4.15.1` 左右的兼容包 | 低版本兼容包 | 新版 WDA 可能在 iOS 13/14 上出现 `xctrunner quited`、`wda started failed`、WDA 主端口断开等运行时问题 |

这里的“低版本兼容包”只是命名分类,不是另一套工具。建议将产物命名为 `wda-ios13-ios14.ipa`,现代包命名为 `wda-ios15plus.ipa`,避免后续分发时装错。

低版本兼容包建议在 macOS + Xcode 15.4 环境单独构建:

```bash
mkdir -p ~/wda-ios13-ios14
cd ~/wda-ios13-ios14
npm init -y
npm i appium-webdriveragent@4.15.1
cd node_modules/appium-webdriveragent
open WebDriverAgent.xcodeproj
```

Xcode 中重点检查:

1. Target 选择 `WebDriverAgentLib` 与 `WebDriverAgentRunner`,均配置同一个开发者 Team。
2. `WebDriverAgentRunner` 的 Bundle ID 建议继续使用项目配置中的 ID,例如 `com.gamehausQaTest.WebDriverAgentRunner.xctrunner`。同一 Bundle ID 可以分别安装到不同设备上,不要求高低版本 WDA 使用不同 ID。
3. `IPHONEOS_DEPLOYMENT_TARGET` 建议设为 `13.0`,确保 iOS 13.6.1 / 14.4.2 能加载运行。
4. 导出 ipa 后只安装到 iOS 13/14 设备; iOS 15+ 继续安装现代 WDA 包。

安装前建议先卸载低版本设备上的旧 WDA:

```bash
tidevice -u <iOS13或iOS14设备UDID> uninstall com.gamehausQaTest.WebDriverAgentRunner.xctrunner
tidevice -u <iOS13或iOS14设备UDID> install wda-ios13-ios14.ipa
```

安装后不要立即进入 MSCA,先单独验证 WDA 是否能稳定运行:

```bash
tidevice -u <设备UDID> wdaproxy -p 8100 -B com.gamehausQaTest.WebDriverAgentRunner.xctrunner

# 另开终端
curl http://127.0.0.1:8100/status
curl http://127.0.0.1:8100/screenshot
```

至少观察 30 秒。若前台日志出现 `xctrunner quited`、`wda started failed`,或 `/status`、`/screenshot` 很快无法访问,说明 WDA Runner 本体仍不适配该设备系统,需要重新检查 WDA 版本、Deployment Target、签名证书和设备信任状态。只有 WDA 能稳定保持可访问后,再回到 MSCA 启动投屏。

### 1.6 设备端准备清单

| 项目 | 操作 | 必要性 |
| :--- | :--- | :--- |
| 信任电脑 | 首次 USB 连接时设备弹窗点"信任" | 必须 |
| 信任开发者证书 | `设置 → 通用 → VPN与设备管理` → 信任 | 必须(安装 WDA 后) |
| 开发者模式(iOS 16+) | `设置 → 隐私与安全性 → 开发者模式` 开启并重启 | iOS 16+ 必须 |
| 关闭屏幕锁密码 | `设置 → 面容ID与密码` 关闭 | 强烈建议(锁屏会阻塞远程控制) |
| 保持亮屏 | `设置 → 显示与亮度 → 自动锁定` 设为"永不" | 建议 |

### 1.7 iOS 17+ 额外要求(Windows)

iOS 17+ 设备需要 go-ios tunnel:

- MSCA 会自动尝试 `--userspace` 模式(无需管理员、无需驱动)
- 若回退到内核模式(管理员提权),需要 **wintun.dll**——**应用已内置于 `bin/ios/wintun.dll`**,与 ios.exe 同目录,go-ios 可直接找到,一般无需手动处理
- 如果你使用的是系统 PATH 中的 ios.exe(而非 MSCA 内置),请把 `bin/ios/wintun.dll` 复制到该 ios.exe 同目录或 `C:\Windows\System32`
- 手动启动 tunnel:以管理员身份运行 `scripts\ios-tunnel.bat`(脚本会自动检查并部署 wintun.dll)

### 1.8 常见问题

| 现象 | 处理 |
| :--- | :--- |
| WDA 启动失败/7 天过期 | 免费证书过期,用 Xcode 重新签名安装 |
| 投屏接口报"Bundle ID 不匹配" | 检查 `backend/config/wda_config.json` 的 `wda_bundle_id` / `wda_bundle_id_pattern` |
| iOS 13/14 WDA 秒退 | 按 1.5 使用低版本兼容 WDA 包;先用 `tidevice wdaproxy` 验证 `/status` 和 `/screenshot` 能稳定访问 |
| iOS 17+ tunnel 启动失败 | 见 1.7;运行日志页面查看具体错误,确认 wintun.dll 就位 |
| 设备显示"不可用" | 连续 3 次投屏失败会被标记,排除证书/信任/tunnel 问题后重启投屏 |

## 二、Android 设备部署

MSCA 的 Android 投屏基于内置 scrcpy-server,由后端通过 ADB 自动推送,**无需在设备上预装任何应用**。只需完成以下设备设置:

### 2.1 通用步骤

1. `设置 → 关于手机` → 连续点击"版本号"7 次,激活开发者选项
2. `设置 → 系统 → 开发者选项` → 开启 **USB 调试**
3. USB 连接电脑,设备弹窗勾选"始终允许此电脑调试"并确认
4. 验证连接:`adb devices` 应显示设备且状态为 `device`(而非 `unauthorized`)

### 2.2 品牌特定设置

| 品牌 | 额外设置 |
| :--- | :--- |
| **华为/荣耀** | 开发者选项中关闭"监控 ADB 安装应用";输入法关闭"安全键盘" |
| **小米/红米** | 额外开启"USB 安装"与"USB 调试(安全设置)";关闭"MIUI 优化"与权限监控 |
| **OPPO/Realme/一加** | 开启"禁止权限监控";关闭"USB 验证";USB 连接方式选"文件传输" |
| **vivo/iQOO** | 开启"安全权限"与"USB 模拟点击";关闭相关电量优化 |
| **三星/魅族/摩托罗拉** | 关闭屏幕锁定;USB 连接方式选"文件传输" |

> 通用建议:控制过程中保持设备亮屏与解锁;部分机型息屏后触控注入会被系统拦截。

### 2.3 常见问题

| 现象 | 处理 |
| :--- | :--- |
| `adb devices` 显示 `unauthorized` | 设备上重新确认 USB 调试授权弹窗;无弹窗时执行 `adb kill-server && adb start-server` 后重插 |
| 设备列表不显示 | 检查 USB 线(部分线仅充电)、驱动(Windows 设备管理器)、`adb devices` 是否可见 |
| 投屏黑屏 | 部分机型对后台录屏限权,确认"USB 调试(安全设置)"类选项已开 |
| 控制无响应 | 小米/OPPO/vivo 需开启上表中的"模拟点击/安全设置"类权限 |
